## Match Scoring Formula

**Score = Σ(factor × weight)** where weights sum to 100:

| Factor | Weight | Logic |
|--------|--------|-------|
| Budget alignment | 30% | buyer.budget_max ≥ property.price AND buyer.budget_min ≤ property.price × 1.20 |
| Location match | 25% | intersection(buyer.locations, property.zone) — exact=25, adjacent=15, same region=5 |
| Property type match | 20% | exact match buyer.property_type ↔ property.property_type |
| Feature match | 15% | len(intersection(buyer.features, property.features)) / len(buyer.features) × 15 |
| Timeline urgency | 10% | buyer.urgency_score × (1 / days_since_listing) normalized |

**Score thresholds**:
- **HIGH** ≥ 80 → auto-trigger deal pack, immediate agent notification
- **MEDIUM** 60–79 → queue for agent review (24h SLA)
- **LOW** < 60 → log to matches table, no automatic action

## Pipeline Stages

```
MATCH ──→ DECISION ──→ DEAL_PACK ──→ SEND ──→ FOLLOWUP ──→ VIEWING
                                                               │
                                                           PROPOSAL
                                                               │
                                                         NEGOTIATION
                                                               │
                                                            CPCV ──→ ESCRITURA ──→ CLOSED
                                                                            ↓
                                                                          LOST (any stage)
```

### Stage Details

| Stage | Entry Condition | Auto-action | SLA |
|-------|----------------|-------------|-----|
| MATCH | Score computed ≥ 80 | Generate deal pack | Immediate |
| DECISION | Deal pack ready | Agent notification | 24h |
| DEAL_PACK | Agent approves match | — | Agent gate |
| SEND | Agent triggers send | Start follow-up timer | Immediate |
| FOLLOWUP | Pack sent, no response | Auto follow-up at 3/7/14/30d | 30 days |
| VIEWING | Buyer confirms viewing | Send prep pack + directions | Immediate |
| PROPOSAL | Agent submits offer | Alert legal team | Immediate |
| NEGOTIATION | Counter-offer received | Flag agent priority_items | 48h |
| CPCV | Offer accepted | Record 50% commission milestone | Legal SLA |
| ESCRITURA | CPCV signed | Record final 50% commission | Notary SLA |
| CLOSED | Deed signed | Investor relations update, learning event | Immediate |
| LOST | Deal fell through | Trigger causal analysis, record reason | Immediate |

## Deal Pack Auto-Generation

**Trigger**: `match_created` event with `score ≥ 80`

**Assembly steps** (crm-orchestrator, claude-opus-4-6, max 8 tool calls):
1. `getPropertyDetails` — fetch full property record + photos
2. `computeMatchScore` — get detailed score breakdown
3. [parallel] `avm-engine` — generate valuation + 6-month forecast
4. [parallel] `followup-generator` — write personalized buyer letter
5. Assemble PDF (property specs, photos, AVM, letter, legal checklist, comparables)
6. Upload PDF to S3 → get CloudFront URL
7. Insert `deal_packs` record with status='ready'
8. Emit `deal_pack_generated` event

**Generation SLA**: < 30 seconds end-to-end
**Fallback**: If AI fails → generate PDF from templates (no personalization)

## Commission Model

**Rate**: 5% of transaction value (standard Portuguese market rate)

**Payment structure**:
- 50% at **CPCV signing** (Contrato-Promessa de Compra e Venda)
- 50% at **Escritura** (public deed / completion)

**Split** (default): 50% agent / 50% company (configurable per deal via `commission_split` JSONB)

**Examples**:

| Property Value | Total Commission | CPCV (50%) | Escritura (50%) |
|----------------|-----------------|------------|-----------------|
| €250,000 | €12,500 | €6,250 | €6,250 |
| €500,000 | €25,000 | €12,500 | €12,500 |
| €1,000,000 | €50,000 | €25,000 | €25,000 |
| €3,000,000 | €150,000 | €75,000 | €75,000 |

**CPCV Tracking**:
- `deals.cpcv_date` set when stage → CPCV
- `deals.escritura_date` set when stage → ESCRITURA
- Commission milestone events emitted to event_history + causal_trace

## Priority Queue (Agent Daily Task List)

`priority_items` table is the agent's work queue, populated by:
- HIGH matches (auto-inserted on match_created)
- Overdue follow-ups (cron daily at 07:00 UTC)
- Deals approaching CPCV/Escritura dates (7-day warning)
- DLQ items needing manual review
- Lead qualification queue (new contacts from WhatsApp overnight)

**Priority 0**: Urgent (CPCV/Escritura same-day, hot lead online now)
**Priority 1–30**: High (overdue follow-ups, HIGH matches not yet reviewed)
**Priority 31–70**: Medium (normal follow-ups, MEDIUM matches)
**Priority 71–100**: Low (routine tasks, cold leads)
