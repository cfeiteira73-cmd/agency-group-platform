# SH-ROS Pipeline Derivation Map
**Wave 12 — Ground Truth Verification**  
**Mandate:** Every KPI must trace to a verified DB column + deterministic formula

---

## How Each Business Primitive is Computed

All computations live in `lib/product/businessPrimitiveEngine.ts`.  
DB: Supabase `deals` + `contacts` tables, filtered by `tenant_id`.

---

### `pipeline_value` (€)

```
SELECT SUM(deal_value) FROM deals
WHERE tenant_id = $org_id
AND fase NOT IN ('post_sale', 'escritura', 'escritura_sell')
LIMIT 1000
```

**Rule:** Uses `deal_value` (NUMERIC). Does NOT use `valor` (display string).  
**Closed detection:** Uses `fase` (portal-compat TEXT), not `stage` enum.

---

### `expected_revenue` (€)

```
SUM(deal.deal_value * deal.probability) for active deals
```

**Rule:** `probability` comes from DB (set at deal creation, default 0.05).  
No hardcoded benchmark probabilities.

---

### `revenue_mtd` (€ month-to-date)

```
SELECT SUM(deal_value) FROM deals
WHERE tenant_id = $org_id
AND fase IN ('post_sale', 'escritura', 'escritura_sell')
AND actual_close_date >= first_of_current_month
```

---

### `commission_mtd` (€)

```
revenue_mtd * 0.05
```

**Source:** `COMMISSION_RATE = 0.05` from `lib/constants/pipeline.ts` — never inline.

---

### `proposals_pending` (count)

```
COUNT active deals WHERE probability >= 0.5
```

**Fixed Wave 12:** Was `contacts.filter(c => c.status === 'negotiating')` — wrong (contact-level).  
Now uses deal-level `probability >= 0.5` = mid-to-late funnel threshold.

---

### `avg_days_to_close` (days | null)

```
AVG(actual_close_date - created_at) for closed deals
```

**Returns `null` when 0 closed deals** — never falls back to market benchmark (210 days).  
UI must display `—` when null.

---

### `close_rate_30d` (0.0–1.0 | null)

```
won_deals_last_30d / total_deals_last_30d
```

**Returns `null` when denominator = 0** — never displays 0% falsely.

---

### `hot_leads` (count)

```
COUNT contacts WHERE tenant_id = $org_id AND lead_score >= 80
```

**Source column:** `contacts.lead_score` (SMALLINT, added by portal_compat_v1 migration).  
Default 0 until scored.

---

### `active_leads` (count)

```
COUNT contacts WHERE tenant_id = $org_id AND status NOT IN ('client', 'lost')
```

**Source column:** `contacts.status` (TEXT, original portal column).

---

### `monthly_target` (€)

```
process.env.ORG_MONTHLY_REVENUE_TARGET ?? 2_000_000
```

**Explicit:** €2M gross = ~€100K/month commission at 5%.  
Set `ORG_MONTHLY_REVENUE_TARGET` in Vercel to override per-org.

---

## Columns Removed (Wave 12)

| Was | Now | Why |
|-----|-----|-----|
| `d.stage` for closed detection | `d.fase` | `stage` is DDL enum; `fase` is the portal-compat TEXT alias that carries the closed stage values |
| `contacts.negotiating` for proposals | `deal.probability >= 0.5` | Proposals are deal-level, not contact-level |
| `clearbit_data.estimated_num_employees ?? 500_000` for `expected_value` | `0` | Employee count ≠ deal value. No deal association at lead level. |
| `monthly_target = 2_000_000` (magic literal) | `Number(process.env.ORG_MONTHLY_REVENUE_TARGET ?? 2_000_000)` | Explicit constant with env override |

## Data Flow Guarantee

```
Supabase deals table
  └─ deal_value  (NUMERIC)          ← arithmetic
  └─ fase        (TEXT)             ← stage filtering
  └─ probability (NUMERIC, 0–1)     ← expected revenue
  └─ actual_close_date (TIMESTAMPTZ) ← MTD / velocity
  └─ tenant_id   (UUID)             ← org isolation
       │
       ▼
businessPrimitiveEngine.getPipeline()
       │
       ├─ pipeline_value      = SUM(deal_value) of active deals
       ├─ expected_revenue    = SUM(deal_value × probability) of active deals
       ├─ revenue_mtd         = SUM(deal_value) of closed deals this month
       ├─ commission_mtd      = revenue_mtd × 0.05
       ├─ proposals_pending   = COUNT(active deals with probability ≥ 0.5)
       ├─ avg_days_to_close   = AVG(actual_close_date - created_at) | null
       └─ close_rate_30d      = won_30d / total_30d | null
```
